1数据从哪来

订阅制 · 本地工具
Claude Code / Codex CLI / Gemini CLI / Kimi / OpenCode 等。由MDM 按系统分别下发 macOS LaunchAgent 或 Windows Task Scheduler 采集脚本，读取本机本用户的聚合用量后上报。
MDM 下发 · Scheduled Task / LaunchAgent
LiteLLM 网关 · API
走公司 LiteLLM 网关的 API key 用量（个人 key 进个人榜，无过期的 agent key 进 Agent 榜）。服务端直连网关管理 API 拉取。

Cursor 企业版 Admin API 的真实 token 与花费，按邮箱归属到个人，无需你做任何操作。
每 2 小时同步
身份 · MDM
按设备序列号反解归属人、邮箱、完整部门路径与头像。机器转手时取「当前在用者」，不会再算到前任头上。

3看板里没有我 / 数据停更了 — 手工补报

通常是MDM 的采集脚本没推到你的机器、或那次执行失败。你可以自己在终端补跑一次，几十秒后刷新看板就能看到自己。

1
macOS 打开「终端」(Terminal)，粘贴并回车
脚本会按你 Mac 的序列号自动认出你是谁，无需填邮箱、部门。需要输入一次开机密码（sudo）。
curl -fsSL https://collector.example.com/tokreport.sh -o /tmp/tokreport.sh && sudo bash /tmp/tokreport.sh manual
2
Windows 打开 PowerShell，使用 IT 提供的 token 补报
Windows 的常驻任务由独立 Scheduled Task 安装；手工补报用 tokreport.ps1，同样只上传聚合 token 计数。
powershell -NoProfile -ExecutionPolicy Bypass -Command "Invoke-WebRequest -UseBasicParsing https://collector.example.com/tokreport.ps1 -OutFile $env:TEMP\tokreport.ps1; & $env:TEMP\tokreport.ps1 -Collector https://collector.example.com -Token <token> -Via manual"
3
看到 tokreport OK 即上报成功
如果提示没装 tokscale，脚本会自动用 npx 临时拉起，首次可能多等十几秒。整段最多跑 3 分钟会自动结束，不会卡住。
4
回到看板刷新，在「个人榜」找自己
前端每 30 秒自动刷新；也可直接 返回看板。仍然没有？大概率是这台机器在MDM里序列号没登记或归属人是空，把序列号发给 IT 核对。
隐私：采集脚本纯只读，只统计 token 数量与花费，绝不读取或上传你的 prompt、对话内容或代码。源码就是上面这个 tokreport.sh，可自行查看。
手工补报的数据，会在个人榜你名字旁标一个「手工」角标 —— 方便和 MDM 自动采集区分、也便于事后核对。正常用不到时，等 MDM 自动采集即可。
